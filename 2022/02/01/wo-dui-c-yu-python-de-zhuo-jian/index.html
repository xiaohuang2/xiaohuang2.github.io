我对 C 与 Python 的拙见

Python 成了近年来炒得非常火的一门编程语言，网上的广告铺天盖地，我身边就有人在学。究竟这门语言怎么样呢，我在阅读了很多大佬的文章之后，在我自己的博客发表这么一篇拙见。

C 语言

编程语言的发展

有这样一句话，“硬件生机器，机器生汇编，汇编生 C，C 生万物”。

从这句话就能看出 C 的地位。

第一阶段

发展的第一阶段，即由硬件诞生的机器码，也就是我们所熟悉的 010101…… 这样的字符串。但人们的目的不就是和计算机沟通并指挥它们做事儿吗？机器码是不可能满足这样的需求了，所以有了第二阶段。

第二阶段

由于机器码太过复杂，人们就把机器码中的语句翻译成了英文单词。我没有学过汇编，也并不了解其中的一些语法，举例就算了。但尽管被翻译过来了，仍然晦涩难学。

第三阶段

高级语言诞生了，C 、C++ 、Java 、Python…… 各种各样的编程语言接踵而来。其中，高级语言又被分为面向过程与面向对象语言两部分。至于它们分别是什么意思，我建议你各学一门后再来体会（其实就是我也不知道）。C 、Pascal 等就属于面向过程，而 Java 、Python 等就属于面向对象。

C 的发展

C 虽然作为老大哥，但在 C 之前，早已有一大批的编程语言。例如 FORTRAN ，它是第一门编译型语言。例如 B ，它可以算 C 的祖先之一。还有很多，大家可以自行 google 来作为了解。

C 的研发者 Dennis Ritchie（丹尼斯 · 里奇）在 B 语言的基础上研发了 C 语言，并在 1978 年与 Brian Wilson Kernighan（布莱恩 · 威尔森 · 柯林汉）一起出版了《The C Programming Language》，书中所树立的标准为 K&R C 标准。1988 年 K&R 又出了该书的第二版，直至今天，这仍是一本经典书籍。1989 年 ANSI 设立的 C 标准被批准，C89 诞生，此后，C90 、C99 ，甚至 C11 都出现了。请自行 google 了解更多。

我的想法

我不建议将 Python 作为你的第一门编程语言，你可以学完 C 再来学 Python ，或者你可以学另外一门比较基础的语言后再来学Python。

Python 最近确实很受欢迎，排名噌噌噌往上涨，看来大家都喜欢简单的东西。简单的也有简单的好处，你可以把心思集中在写程序上，而不是改语法上。Python 也被吹到了各大行业领域，如人工智能，web 开发等等，其实也并没有那么夸张。

对于咱们来说，学 Python 就像是学了一个小工具，要不然 Python 怎么是脚本语言呢。学会了 Python ，日常的一些工作都可以通过写一些自动化的脚本来方便。Python 也有一些进阶用法，之前我学网络安全时有了解，什么爬虫啊，多线程啊等等，当然那都是后话了。

我之后肯定也会去学 Python 的，只是之前学校里有个不同班的同学用 Python 写了个小程序交到学校宣传的比赛去了，而我那时连 C 都还没怎么学，也就没写，看得心里很不爽，然后就又开始学 C 了，其实之前都好长一段时间没有学过了，毕竟学生嘛，电脑使用还是有限制。

至于更多有关 Python 的建议，可以上b站搜一搜，很多大佬都发表了自己的看法，如 codesheep。

没有最好的语言，只有最适合自己的。
